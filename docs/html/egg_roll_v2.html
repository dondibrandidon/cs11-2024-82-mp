tilt(degree: str, moves_left: int) → tuple[list[str], int, tuple[str, ...], bool]¶

The main logical interaction with the player! Simulates a tilting of the board, which usually moves eggs.

Parameters:

  • degree (str) – A character in “fbrl” which determines direction.

  • moves_left (int) – Moves Player has left, used to calculate points.

Raises:

ValueError – Raised when :param: degree is not in “fbrl.”

Returns:

Returns debugging logs, points gained or lost, array for grid animation, and whether there are any eggs left.

tuple[list[str], int, tuple[str], bool]

class egg_roll_v2.Player(level_file: TextIOWrapper, freedom: dict[str, str] = {'backward': 'bB', 'forward': 'fF', 'leftward': 'lL,', 'rightward': 'rR'}, is_debug: bool = False)¶

Bases: object

This class is the controller and view handling the gameplay.

Parameters:

  • level_file (TextIOWrapper) – The level file to be read and processed

  • freedom (dict[str, str], optional) – The dictionary defining accepted inputs, defaults to example control scheme (“fFbBrRlL”).

  • is_debug (bool, optional) – Makes debugging more convenient if True, defaults to False

Variables:

  • degrees (dict[str, str]) – This turns the player input into the characters tilt() can process

  • valids (str) – String containing all valid inputs from degrees

  • file_name (str) – The name of the level file

  • moves_left (int) – Decrements with each successful character or “undo” input, treated as infinite if set to -1

  • max_moves (int) – The number of moves prescribed in level file

  • current_level – The main Level instance interacted with

  • points (int) – All points gained by so far

  • game_end (bool) – Starts False, the Game Over screen is displayed once True

  • past_moves (list[str]) – Array of symbols to denote past successful inputs

  • past_levels (list[Level]) – Array of past Level states for undo-ing

  • past_points (list[int]) – Array of past points for undo-ing

  • current_input (str) – The input for processing to tilt()

  • debug (bool) – Enables printing of debug info and disables time.sleep() if True

Raises:

ValueError – Raised for Level with grid that doesn’t follow a TileSet

_set_current_input() → None¶

This prints the prompt asking for user input.

_exit_scenario() → None¶

This is for the scenario of an exit input. Simple sets game_end to True.

_undo_scenario() → None¶

This is for the scenario for an undo input. Note that the last state saved is the current state.

If there are past states to undo to, this method sets the second to the last Level and int in past_levels and past_points to be current_level and the current points, respectively.
